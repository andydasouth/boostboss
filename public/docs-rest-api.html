REST API integration

For AI bots running server-side — Discord, Telegram, Slack, or any platform where you can't ship a JS bundle to the user. One POST /v1/ad-request from your bot's response handler. Helper libraries map the response to each platform's native message format.

This page is being expanded.

The full deep-dive (every endpoint, every parameter, retry semantics, helper-lib API for Discord/Telegram/Slack) is in progress. For the time being, the comprehensive API reference covers the underlying endpoints in detail.

The one endpoint you'll call

POST https://api.boostboss.ai/v1/ad-request
Authorization: Bearer ${API_KEY}
Content-Type: application/json

{
  "format": "embed",                  // "embed" | "card" | "text"
  "context": "user asked about Stripe billing",
  "platform": "discord",              // optional, helps shape response
  "user_region": "US"                  // optional, ISO region
}

Response includes headline, body, image_url, click_url, impression_url, and a disclosure_label. Render whichever fields fit your platform.

Discord example

// Inside your Discord bot's response handler:
const ad = await fetch('https://api.boostboss.ai/v1/ad-request', {
  method:  'POST',
  headers: { 'Authorization': `Bearer ${API_KEY}` },
  body:    JSON.stringify({
    format:   'embed',
    context:  userQuery,
    platform: 'discord'
  })
}).then(r => r.json());

message.reply({
  content: aiResponse,
  embeds:  [ad.toDiscordEmbed()]   // from @boostbossai/lumi-discord
});

Helper libraries

Each helper library maps the generic ad response to the target platform's native format. Same source data; different output shape.

  • @boostbossai/lumi-discord — toDiscordEmbed(), toDiscordComponents()
  • @boostbossai/lumi-telegram — toTelegramMessage(), toTelegramInlineKeyboard()
  • @boostbossai/lumi-slack — toSlackBlocks(), toSlackAttachment()

Impression tracking

After your bot sends the message containing the ad, hit the impression_url we returned:

await fetch(ad.impression_url);   // fire-and-forget; no auth needed

The helper libraries fire this automatically when you call their send() wrapper. If you're using raw fetch + message.reply, fire it yourself after the message has actually been delivered.

Click tracking

The click_url we return is already a tracking redirect — link your CTA button to it directly. Don't substitute the underlying advertiser URL; that bypasses tracking and breaks revenue attribution.

Rate limits

1,000 requests/minute per publisher. Burst capacity above the steady rate is allowed for short windows. Email support@boostboss.ai if you're approaching the cap and need it raised.

Error codes

HTTPCodeMeaning
200—Ad available; render and track impression.
204NO_FILLNo advertiser matched. Send your AI response without an ad.
401BBX_AUTHBearer token invalid or revoked.
429BBX_RATE_LIMITPer-minute cap exceeded.
5xx—Upstream issue. Treat as no-fill; do not retry on the bot path.

Testing

Use a sandbox API key from the dashboard (prefix sk_test_) — sandbox always returns a fixed creative regardless of context. No real demand-side calls; impressions don't count.
